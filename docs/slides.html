Reproduction study on black box LLM unlearning

Teaching a language model to forget, with no retraining. Then turning the same idea into answer correction.

A faithful reimplementation of a RAG based unlearning method, an analysis of where it fails, and a new direction. From suppressing answers inside a system you own, to correcting what public models say about an entity.

Reproduction, critique, prototype Base paper: arXiv:2410.15267 Code: github.com/xuwudawei/rag-unlearning
Motivation02 / 13

Why make a trained model forget?

Regulation, copyright, and safety all demand removing specific knowledge from a model that is already deployed. Retraining a frontier model for every request is not affordable.

The constraint

Commercial models expose only an interface. No weights, no gradients. A method has to work from the outside, without training.

The gap

Most unlearning methods fine tune the weights and risk damaging the rest of the model. They cannot be applied to closed models at all.

The idea under test

Use retrieval as the control surface. Change what the model reads, rather than what the model is.

Base paper03 / 13

When Machine Unlearning Meets RAG

Wang, Zhu, Ye, Zhou. City University of Macau. arXiv:2410.15267. No official code was released.

Mechanism

Inject a knowledge entry k = P + Q into the retrieval store. P makes the entry match questions about the target. Q is a confidentiality clause that makes the model refuse.

Access model

Interface only. The provider controls retrieval and never touches the weights. It works on closed models, where fine tuning based methods cannot be applied.

  • Framed as constrained optimization. Maximise forgetting over the knowledge store, subject to the entry being retrievable.
  • This is behavioural unlearning. It changes the output, not the parameters. That distinction drives the critique three slides on.

Method04 / 13

Algorithm 1, reproduced faithfully

Construct the entries

Q is written by a helper model and requeried until the frozen target refuses. P is a set of aspect descriptions that the target writes about itself, which lifts retrieval rank. One entry per aspect.

Retrieve and generate

A hybrid retriever combines keyword and embedding search. The prompt template joins the instruction, the question, and the retrieved knowledge.

USR

Unlearning success rate. A judge model scores forgetting from the before and after answers.

ROUGE L recall

Overlap of the unlearned answer with the original. Lower means more forgetting.

Min K membership

Inference for leftover memorisation. Needs token probabilities, so it runs on the local model.

Critique06 / 13

Where the method breaks

  • Suppression is not forgetting. The knowledge stays in the weights and the answer is only intercepted at retrieval. A recent position paper makes exactly this complaint about the word unlearning.
  • Fragile to injection, by their own Table XII. An attacker who leaks part of the entry and appends an override drives success down to 20.9 percent.
  • Retrieval is a single point of failure. A paraphrase, a translated question, or a multistep query can miss the entry entirely.
  • Excess refusals are never measured. The paper tests MMLU and ARC, which contain none of the forgotten topics, so damage to neighbouring topics stays invisible.
Success falls from 99 percent to 20.9 percent under attack The authors leave the defense to future work

Contribution 107 / 13

A defense that does not rely on obedience

The paper's forgetting only holds while the model obeys the retrieved clause. We add two layers that do not depend on the model obeying anything.

Input guard

Detect override and injection patterns, then reassert the constraint before the model answers.

Output gate

Independent of the model. If a question about a forgotten topic yields target content, replace it with a refusal after generation, so an injection cannot leak it.

Honest negative result

On a well behaved model the baseline never collapsed, since it refused on its own. So the guard showed no gain here and it added excess refusals, from 40 to 60 percent. Its value needs a genuinely jailbreakable target. Reported, not hidden.

Key insight08 / 13

Three problems, one mechanism

A retrieval augmented model is only as trustworthy as the context it retrieves. Suppression, influence, and attack are the same lever pulled from different ends.

Unlearning

The owner injects entries to suppress an answer.

Answer optimisation

An outside party shapes the web to influence an answer.

RAG poisoning

An attacker injects sources to corrupt an answer.

So the paper we reproduced is the defensive half of a mechanism whose corrective half is a real and unbuilt product. Measuring, then fixing, what models say about an entity.

Contribution 209 / 13

Answer Estimator. Invert the optimization.

Rather than suppress an answer inside a store you own, measure what the model says about an entity, find which sources drive it, and predict the cheapest legitimate correction. This ran on live DeepSeek.

1 · Measure

Probe the model several ways. The false claim appeared 80 percent of the time.

2 · Attribute

Remove one source at a time to see which sources drive the claim.

3 · Intervene

Predict the effect of each legitimate action and recommend the cheapest.

Result on the worked example

Publishing a corrective statement on your own site takes the claim rate from 80 percent to 0 percent at the lowest cost, ahead of third party takedowns. Unlike a monitoring dashboard, this is causal and predictive. It says what to do and the expected result.

Validity11 / 13

What I do not claim

  • Reproduction scale. Five topics against the paper's hundred. The effect reproduces. This is not a full replication.
  • Membership inference runs only on the local open model, since interfaces do not expose input token probabilities.
  • The guard's benefit is unproven and it currently trades excess refusals. It needs a jailbreakable target to show value.
  • Estimator attribution is a first version and noisy. Five probes and a model that varies run to run make the per source numbers wobble. It needs Shapley values, more probes, and repeated sampling with confidence intervals.
  • A controlled source set, not the live web. A faithful proxy. Real citation grounded engines such as Perplexity and ChatGPT Search are the next integration.

Future work12 / 13

Open questions worth a paper

Harden attribution

Shapley value source influence with confidence intervals, checked against ContextCite.

Live grounded engines

Causal attribution on the real citations returned by Perplexity and ChatGPT Search.

Verifiable unlearning

Removal backed by a certificate. Adversarial probes, membership evidence, and an audit trail.

Answer integrity monitor

Detect poisoning from the gap between the predicted answer and the observed one.

Summary

Reproduced a black box unlearning method on a live model, showed where it fails, and prototyped an inversion from suppression to causal, predictive answer correction, grounded in ContextCite and influence function theory.
